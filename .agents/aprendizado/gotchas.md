# Registro de Aprendizado — Gotchas do Sistema

> Este arquivo registra padrões problemáticos que o sistema aprendeu a evitar.
> O Orchestrator e todos os agentes **consultam este arquivo antes de gerar qualquer output** para garantir que erros corrigidos não se repitam.
>
> **Manutenção:** Atualizar ao final de sessões onde houve correção de estilo, tom ou fato.
> **Limite:** máximo de 50 registros ativos. Curar e arquivar os obsoletos trimestralmente.

---

## Como registrar um novo Gotcha

```
## GOTCHA-[N]
- **Padrão problemático:** [o que foi feito de errado]
- **Contexto:** [onde se aplica — ex: copy de carrossel, proposta, email]
- **Por que é um erro:** [a razão clara]
- **Alternativa correta:** [o que fazer no lugar]
- **Ocorrências:** [número de vezes que aconteceu]
- **Registrado em:** [data]
- **Fonte:** [quem corrigiu — ex: Leo, 2026-07-09]
```
