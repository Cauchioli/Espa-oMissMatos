# /authority-engine — Arsenal de Autoridade (Vértice OS)

Um especialista precisa publicar a mesma ideia durante anos sem parecer repetitivo. O Authority Engine pega uma única ideia central e a expande em todos os formatos de autoridade possíveis — com posições fortes e takes originais.

## Filosofia

Autoridade não é quantidade de posts. É uma ideia repetida com profundidade crescente em formatos diferentes. Quem domina um conceito e o articula em múltiplas camadas vira referência inevitável.

## Contexto Necessário (leitura automática)

- `_memoria/empresa.md` — nicho, credenciais, posicionamento
- `_memoria/preferencias.md` — tom de voz, o que evitar
- `identidade/messaging/tese-framework.md` — se existir
- `identidade/messaging/big-ideas.md` — se existir

## Entradas (via conversa se necessário)

- `ideia-central` — o conceito que o usuário quer dominar (ex: "a consulta jurídica é um diagnóstico")
- `ponto-de-vista-controverso` — o que o usuário acredita que vai contra o senso comum do nicho

## Saídas — Pasta `autoridade/[slug-da-ideia]/`

### 1. opiniao-forte.md
Manifesto de 400-600 palavras sobre a tese. Estrutura:
- Afirmação da posição (clara, sem hedge)
- Por que o mercado está errado (o contraste)
- A lógica por trás da posição (3 argumentos)
- O convite (para quem concorda e quer ir mais fundo)

### 2. contrarian-takes.md
3 takes que contradizem o senso comum do nicho. Para cada um:
- O take (1 frase provocativa)
- O argumento (3-5 linhas)
- Onde usar (post, thread, palestra, entrevista)

### 3. tese-framework.md
O framework proprietário da ideia. Deve ter:
- Um nome memorável (2-4 palavras)
- 3 a 5 etapas ou elementos
- Um diagrama descritivo (texto que descreve o visual para o designer)
- Como o framework resolve o problema do avatar

### 4. artigo-longo.md
Artigo de autoridade completo (1.200-2.000 palavras) com:
- Título (3 opções)
- Lide (gancho nos primeiros 2 parágrafos)
- Desenvolvimento com subtítulos (H2/H3)
- Exemplos concretos do nicho
- Conclusão com CTA suave (ex: "se quiser aprofundar, fala comigo")

### 5. roteiro-palestra.md
Estrutura de palestra de 20 minutos:
- Abertura (gancho e credencial — 2 min)
- O problema (o que está errado no mercado — 5 min)
- A solução (o framework — 8 min)
- Prova (case ou dado — 3 min)
- Encerramento + CTA (2 min)
Com sugestões de slides para cada seção.

### 6. roteiro-podcast.md
Roteiro para quando o usuário for convidado de um podcast:
- A apresentação ideal (como pedir para o host te apresentar)
- 5 histórias/exemplos prontos para usar nas respostas
- As 10 perguntas que o usuário quer que façam (e as respostas)
- O CTA de saída (o que deixar para o ouvinte)

### 7. outline-livro.md
Estrutura de um futuro livro sobre a ideia:
- Título (3 opções)
- Premissa central (1 parágrafo)
- Capítulos (8-12) com:
  - Título do capítulo
  - Argumento central
  - Exemplo ou história-âncora
- Epílogo e CTA do livro

## Princípios

1. Takes fortes > conteúdo neutro. Quem não polariza não é lembrado.
2. Usar exemplos do nicho real — nunca metáforas genéricas de esportes ou guerra.
3. O framework deve ter nome. Nome sem nome não escala.
4. O artigo longo é o ativo mãe — os outros formatos são derivados dele.
